# 车牌识别应用文档

## 1. 应用介绍

本应用基于 **MaixCAM** 开发，采用 **PP-OCR** 模型实现车牌识别功能。该应用通过 STM32 发送触发信号后，在 4 秒内进行车牌识别。识别成功后，车牌信息将通过 UART 返回。应用具有去抖动、重复识别抑制等功能，适用于出入口道闸等场景。

## 2. 应用部署方法

### 2.1 硬件环境

- **MaixCAM**：设备运行本应用所需的硬件平台。
- **STM32**：触发车牌识别的外部设备，通过串口与 MaixCAM 进行通信。

### 2.2 软件环境

- **MaixPy**：支持运行 Python 程序的固件。
- **PP-OCR**：车牌识别模型，已嵌入在 MaixCAM 中。

### 2.3 配置文件

- **app.yaml**：包含应用的基本配置信息。
- **main.py**：主要的 Python 应用程序代码。

## 3. 应用使用方法

1. **启动应用**：
    - 将应用文件上传到 MaixCAM 开发平台。
    - 确保设备已经连接到外部串口触发设备（STM32）。
    - 在设备的应用管理界面启动应用。
    
2. **车牌识别**：
    - 通过 STM32 发送 `REQ_OCR` 触发信号。
    - MaixCAM 在 4 秒内执行车牌识别。
    - 识别结果会通过 UART 返回给 STM32。
    
3. **触摸退出**：
    - 按屏幕右上角的 `>` 按钮可以退出应用。

## 4. 原理和实现介绍

### 4.1 车牌识别原理

- 采用 **PP-OCR** 模型进行车牌字符的识别。
- 系统先对图像进行预处理（缩放、格式转换），然后送入模型进行预测。
- 识别结果会根据车牌格式过滤，并通过串口回传给外部设备。

### 4.2 主要功能实现

- **事件触发模式**：仅在收到 STM32 触发信号后进行车牌识别，减少不必要的计算。
- **去抖动机制**：在 3 秒内不会重复发送相同车牌信息，减少误报。
- **状态显示**：通过屏幕显示当前状态，如 `IDLE`、`ARMED IN`、`ARMED OUT`。
